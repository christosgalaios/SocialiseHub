## Feedback #1 — Platform Connect & Scrape Refactor

Reviewing your uncommitted changes to meetup.ts, eventbrite.ts, headfirst.ts, and ServicesPage.tsx.

**Good decisions:** API calls over DOM scraping for Meetup/Eventbrite, consistent 120s polling, navigating to login pages directly, .co.uk for Eventbrite.

### Issues to fix:

1. **Meetup GraphQL injection risk** — In `meetupScrapeSteps`, `groupUrlname` is interpolated directly into the query string. If urlname contains quotes or special chars, it breaks. Use GraphQL variables:
   ```js
   body: JSON.stringify({
     query: 'query($urlname: String!) { groupByUrlname(urlname: $urlname) { events(first: 50) { edges { node { id title dateTime eventUrl venue { name } } } } } }',
     variables: { urlname: "${groupUrlname}" }
   })
   ```

2. **Headfirst scrape uses a blind 3-second setTimeout** — Fragile. Poll for event links instead:
   ```js
   // Replace: await new Promise(r => setTimeout(r, 3000));
   // With:
   for (let i = 0; i < 10; i++) {
     if (document.querySelectorAll('a[href*="/events/"]').length > 0) break;
     await new Promise(r => setTimeout(r, 1000));
   }
   ```

3. **Headfirst org name extraction is brittle** — The regex `^(.+?)Organisation` on sidebar text will break if nav changes. Just use the org link's own text content instead.

4. **Eventbrite scrape has no auth guard** — If user isn't logged in, you silently return `[]`. Return `{ error: 'Not authenticated' }` instead so the caller can distinguish "no events" from "not logged in."

5. **`waitForSelector: 'body'` is a no-op** — Body always exists. Fine for API-based steps, but just noting it offers no readiness signal. Consider removing or using a meaningful selector.

---

## Acknowledged

1. **Meetup GraphQL injection** — Fixed. `meetupScrapeSteps` now uses `query($urlname: String!) { groupByUrlname(urlname: $urlname) ... }` with `variables: { urlname }` instead of string interpolation.
2. **Headfirst blind setTimeout** — Fixed. Replaced with polling loop: checks for `a[href*="#/events/"]` count > 2 every 500ms, with 15s timeout.
3. **Headfirst org name** — Fixed. Now reads directly from the org link's `textContent` instead of regex-parsing surrounding sidebar text.
4. **Eventbrite auth guard** — Fixed. Scrape now checks `/api/v3/users/me/` first and returns `{ error: 'Not authenticated' }` if not logged in. All three `*-client.ts` `fetchEvents()` methods now check for error objects and throw instead of silently returning `[]`.
5. **`waitForSelector: 'body'`** — Acknowledged as a no-op. Left as-is since the evaluate scripts handle their own readiness via API calls/polling. The `waitForSelector` step just ensures the page has loaded enough to execute JS.
